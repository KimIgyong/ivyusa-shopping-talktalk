## 1. 시작하기

### 1.1 콘솔 접속
- 운영자·관리자 콘솔(`apps/web`)은 브라우저로 접속합니다. 개발 환경 기준 웹 `:5173`, API `:3000`(`/api/v1/docs`에 Swagger).
- 로그인은 이메일 + 비밀번호(JWT 인증). 액세스 토큰 15분, 리프레시 7일.

### 1.2 시드 계정 (최초 로그인 시 비밀번호 변경 필수)
| 역할 | 계정 | 비밀번호 |
|---|---|---|
| 시스템 관리자 | `admin@amoeba.group` | `amb2026!@` |
| 테넌트 마스터(ivyusa) | `dev@amoeba.group` | `amb2026!@` |

> 두 계정 모두 `must_change_password=1` 상태이므로 첫 로그인 후 비밀번호를 바꿔야 합니다.

### 1.3 역할·권한 (RBAC)
권한은 **직급 × 라벨** 매트릭스로 결정됩니다.
- **직급(rank)**: master · director · manager · staff
- **라벨(label)**: consult(상담) · accounting(정산) · operations(운영)
- **시스템 관리자**: super · admin (테넌트 교차)

메뉴·기능 노출은 이 권한에 따라 달라지며, ACL 소유자 가시성 정책(POL-019)이 기능 권한 위에 적용됩니다.

### 1.4 콘솔 메뉴 구조
| 메뉴 | 경로 | 용도 |
|---|---|---|
| 대시보드 | `/` | 상담 지표 개요 |
| 라이브챗 | `/live-chat` | 실시간 상담·인계 처리 |
| 상담 이력 | `/history` | 지난 대화 조회·필터 |
| AI 설정 | `/ai-setting` | 페르소나·규칙·시나리오·엔진·검열 |
| 지식 | `/knowledge` | KB 소스·문서 관리 |
| 고객 | `/customers` | 고객 캐시·등급 |
| 캠페인 | `/campaigns` | 캠페인 관리 |
| 사용자 | `/users` | 사용자·권한 |
| 환경설정 | `/settings` | 테넌트 설정 |
| 관리자 | `/admin/*` | 플랫폼 관리(테넌트·AI엔진·감사) |

## 2. 고객 위젯 대응 흐름

고객이 Shopify 스토어프론트의 채팅 위젯에 진입하면 다음 순서로 대응됩니다.

### 2.1 비로그인 진입 → 시나리오 챗봇
위젯은 먼저 **시나리오 버튼 메뉴**를 노출합니다. 기본 6개 액션:

| 버튼 액션 | 설명 |
|---|---|
| `delivery_status` | 배송 상태 안내 |
| `cancel_refund` | 취소 / 환불 |
| `product_help` | 상품 도움말 |
| `contact_support` | 상담원 연결 |
| `affiliate` | 제휴 |
| `my_orders` | 내 주문 |

> ⚠️ 여기서 "시나리오 챗봇"은 **버튼 기반 빠른 메뉴(quick-reply)**로, 문의 의도를 각 액션으로 라우팅합니다. 다분기 대화 트리 엔진이 아니라, 정형 문의를 빠르게 분류·안내하는 메뉴입니다. 실제 자유질문 답변은 RAG가 담당합니다.

### 2.2 본인확인 게이트 (개인정보가 필요한 경우)
고객이 주문·배송처럼 개인정보가 필요한 질문을 하면, 백엔드의 **의도 분류기**가 `needsOrderData=true`로 판단합니다. 이때 고객이 비로그인(customerId 없음) 상태이면 시스템이 안내합니다:

> "주문을 조회하려면 본인 확인이 필요합니다. 로그인하거나 비회원 주문 조회를 이용해 주세요." (세션 언어에 따라 en/es/ko)

- 비회원 주문 조회는 **15분당 최대 5회**로 제한됩니다(Redis 레이트리밋).

### 2.3 정형 답변으로 불충분 → RAG AI 챗봇
자유 질문이거나 시나리오로 해결되지 않으면 **RAG 지식기반 AI 챗봇**이 응답합니다.
1. 테넌트 KB에서 관련 문서 검색(활성 + 테넌트 스코프)
2. 검색 근거로만 답변 생성, **출처(citation) 첨부**
3. **신뢰도(confidence)** 산정
4. **검열 게이트** 통과 후 고객에게 전달

### 2.4 지식 범위 밖 → 상담원 인계
신뢰도가 낮거나(< 0.45) 검열에 차단되면 상담원에게 넘깁니다. 상세 조건은 [5장](#5-상담원-인계에스컬레이션-조건과-시점) 참조.

### 2.5 대화 상태 전이
```
ai_active  ──(에스컬레이션)──▶  waiting  ──(상담원 인수)──▶  agent  ──(종료)──▶  ended
```

### 2.6 위젯 화면 구성 (하단 탭)
고객 위젯은 채팅만이 아니라 하단 탭으로 세 개의 화면을 제공합니다.

| 탭 | 아이콘 | 용도 |
|---|---|---|
| 알림 (Notifications) | 🔔 | 결제/배송/이벤트/리뷰 알림 확인, 안읽음 배지·읽음 처리 |
| 채팅 (Chat) | 💬 | 시나리오 버튼 + RAG AI + 상담원 (본 장 앞부분 내용) |
| 주문 (Orders) | 📦 | 결제/배송/문의 하위탭, 주문 상세·배송 추적, 리뷰 작성 |

### 2.7 주문(Orders) 탭
- **하위탭**: 결제(payments = 전체 결제 주문) · 배송(shipping) · 문의(inquiries = 취소·환불·반품 관련).
- **본인확인 게이트**: 비로그인 상태에서는 로그인/비회원 조회를 요구하는 인증 게이트(AuthGate)가 표시됩니다. 인증 후 주문 목록·상세를 조회합니다.
- **주문 상세**: 배송 단계 추적 표시(Confirmed → In Transit → Delivered → Review, POL-014 상태 매핑). "이 주문 문의하기"를 누르면 해당 주문번호로 채팅이 연계됩니다.

### 2.8 리뷰 작성
- 배송이 완료(Delivered → Review 단계)된 주문 항목에 대해 **별점(1–5) + 후기 텍스트**를 남길 수 있습니다.
- 세션 토큰 기반으로 저장되며, 제출 후 감사 메시지를 표시합니다.

### 2.9 알림(Notifications) 탭 & 수신 환경설정
- **필터**: 전체 / 결제(payment) / 배송(shipping) / 이벤트(event) / 리뷰(review). 안읽음 항목은 점으로 표시되고, 클릭하면 읽음 처리됩니다. 하단 탭의 알림 아이콘에는 **안읽음 개수 배지**가 표시됩니다(99+ 상한).
- **수신 환경설정(PreferencesPanel)**: 채널 × 카테고리 매트릭스로 알림 수신 여부를 토글합니다.
  - 채널: `in_app`(항상 켜짐·끌 수 없음) · `email` · `sms` · `web_push`
  - 카테고리: `payment` · `shipping` · `event` · `review`
  - **CCPA 옵트아웃** 링크: `in_app`을 제외한 모든 채널·카테고리 수신을 한 번에 끕니다.
- **재입고 알림(back-in-stock)**: 품절 상품에 대해 **게스트도 구독 가능**합니다(`POST /restock/subscribe`, 채널 지정). 재입고 시 알림이 발송됩니다.

### 2.10 개인정보 동의 (Consent)
- 위젯 진입 시 동의 배너(ConsentBanner)로 CCPA/GDPR 고지·동의를 받습니다. 개인정보가 필요한 기능(주문 조회 등)은 동의·본인확인 이후 이용됩니다.

## 3. 시나리오 챗봇 설정

**위치**: AI 설정(`/ai-setting`) → "시나리오 버튼" 섹션 · **권장 권한**: 마스터/디렉터 (consult)

### 3.1 버튼 편집
각 버튼은 다음 필드를 가집니다.

| 필드 | 설명 |
|---|---|
| `id` | 버튼 식별자 (미지정 시 `btn_N` 자동 부여) |
| `label` | 화면 표시 문구 (최대 60자) |
| `action` | 라우팅 액션 (아래 목록 중 선택) |
| `enabled` | 노출 여부 (false면 위젯에 표시 안 됨) |

선택 가능한 `action`: `delivery_status`, `cancel_refund`, `product_help`, `contact_support`, `affiliate`, `my_orders`, `message`.

### 3.2 동작 방식
- 관리자가 저장한 버튼 집합은 위젯이 공개 엔드포인트 `GET /ai-config/scenario`로 조회하며, **enabled=true인 버튼만** 내려갑니다.
- 저장된 설정이 없으면 시스템 기본 6버튼이 사용됩니다.
- 위젯은 조회 실패 시에도 하드코딩된 기본 버튼을 표시해 **항상 메뉴가 보이도록** 폴백합니다.
- 라벨은 위젯 다국어(en/es/ko)로도 폴백 번역됩니다.

### 3.3 설정 팁
- 자주 들어오는 정형 문의 순서로 버튼을 정렬하세요(위/아래 이동 지원).
- 불필요한 버튼은 삭제보다 `enabled=false`로 숨기면 재활성화가 쉽습니다.
- `contact_support`는 항상 활성 상태로 두어 고객이 즉시 상담원을 요청할 수 있게 하는 것을 권장합니다.

## 4. RAG 지식기반 AI 챗봇 설정

**위치**: AI 설정(`/ai-setting`) · **권장 권한**: 마스터/디렉터

RAG 챗봇의 응답 품질은 세 가지 설정 + 지식베이스로 결정됩니다.

### 4.1 봇 페르소나 (Persona)
- "봇 페르소나" 섹션의 텍스트 영역에 봇의 말투·태도·원칙을 서술합니다.
- 이 텍스트는 RAG 시스템 프롬프트의 앞부분에 주입됩니다.
- 기본값: *"You are IVY USA's friendly, concise customer-support assistant. Be helpful and accurate, and only answer from the provided knowledge."*

### 4.2 응답 규칙 (Response Rules)
- 여러 줄의 규칙을 추가/삭제할 수 있습니다(예: "가격은 단정하지 말 것", "환불은 정책 문서를 인용할 것").
- 저장 시 공백 규칙은 자동 제거됩니다.
- 규칙은 페르소나 다음에 목록 형태로 프롬프트에 주입됩니다.

### 4.3 기능별 AI 엔진 (AI Functions)
기능별로 사용할 AI 엔진을 지정합니다. 기능 키: `chat`, `rag`, `summary`, `assist`, `moderation`.
- `AiGatewayService`가 `(테넌트, 기능)` → `tenant_ai_settings → ai_engines`로 엔진을 해석합니다.
- 어댑터: **스텁**(키 없이 동작, 개발/폴백) · **Anthropic**(키 설정 시).
- 엔진 장애 시 자동으로 스텁으로 강등되어 응답이 끊기지 않습니다.

### 4.4 RAG 검색·답변 로직 (참고)
- 검색은 활성(`active=1`) + 테넌트 스코프(`tenant_id` 일치 또는 NULL) 문서를 대상으로 합니다.
- **Knowledge Store 소스가 Google Drive 소스보다 우선** 정렬됩니다(POL-013).
- 신뢰도는 검색된 문서 수에 따라 산정됩니다. 실제 공식: 문서가 없으면 `0.2`, 있으면 `min(0.95, 0.5 + 문서수 × 0.12)` (예: 1건 0.62, 4건 이상 최대 0.95).
- 답변은 "컨텍스트에서만 답하고, 부족하면 상담원 연결" 원칙으로 생성되며 출처를 함께 반환합니다.

> 🟡 현재 검색기는 **키워드 기반**으로, 벡터 임베딩 검색은 로드맵입니다. 따라서 **KB 문서의 제목·본문에 고객이 쓸 법한 키워드를 충분히 포함**시키면 검색 적중률이 올라갑니다.

### 4.5 설정 체크리스트
- [ ] 페르소나에 브랜드 톤·금지사항 명시
- [ ] 핵심 정책(배송/취소/환불)마다 KB 문서 등록 + 활성화
- [ ] 응답 규칙에 "근거 없으면 단정 금지" 포함
- [ ] `rag` 기능 엔진이 원하는 프로바이더로 지정됐는지 확인

## 5. 상담원 인계(에스컬레이션) 조건과 시점

RAG 답변 후, 시스템은 다음 로직으로 인계 여부를 판단합니다.

### 5.1 자동 인계 트리거
| 조건 | 동작 |
|---|---|
| **신뢰도 < 0.45** (`ESCALATION_CONFIDENCE`) | 답변에 "상담원 연결을 도와드릴까요?" 문구를 덧붙이고 대화를 `waiting`으로 표시 |
| **검열 게이트 차단**(BLOCKED) | 답변을 노출하지 않고 "상담원에게 연결해 드리겠습니다" 안내 후 `waiting` 처리 |
| **고객이 상담원 요청** | 위젯의 상담원 연결(`contact_support`) 또는 명시적 에스컬레이션으로 `waiting` 처리 |

### 5.2 처리 순서 (내부 동작)
1. 사용자 메시지 저장 · 대화 로그/CJM 이벤트 발행
2. 의도 분류 — 주문 데이터 필요 시 본인확인 게이트
3. RAG 답변 생성 (검색 → 생성 → 신뢰도)
4. **검열 게이트**(필수, fail-safe 차단)
5. `신뢰도 < 0.45` **또는** `검열 차단` → 에스컬레이션
6. 대화 상태 `waiting`으로 변경 → 상담원 대기열에 노출

### 5.3 임계값 조정
- 현재 인계 신뢰도 임계값은 코드 상수(`ESCALATION_CONFIDENCE = 0.45`)로 정의되어 있습니다.
- 🟡 화면에서 직접 조정하는 UI는 아직 없습니다. 값 변경이 필요하면 개발팀에 요청하세요(로드맵: 관리자 설정화).

### 5.4 인계 시 고객 안내 문구 (다국어)
세션 언어에 따라 자동 현지화됩니다.
- 한국어: "이 문제를 도와드릴 상담원에게 연결해 드리겠습니다." / "상담원에게 연결해 드릴까요?"
- 영어/스페인어도 동일 의미로 제공.

## 6. 상담원 라이브챗 콘솔

**위치**: 라이브챗(`/live-chat`) · **권장 권한**: consult 라벨 보유자

### 6.1 세션 대기열
- `waiting`(대기) · `agent`(상담 중) 상태의 대화가 목록에 표시되며, 각 항목에 마지막 메시지가 함께 보입니다.
- 페이지네이션으로 조회합니다.

### 6.2 대화 인수 (Accept)
- 대기 중 대화를 **인수**하면 배정(assignment)이 생성되고 대화 상태가 `agent`로 바뀌며 담당 상담원이 지정됩니다.

### 6.3 AI 브리핑 (Briefing)
- 대화를 이어받을 때 AI가 지금까지 대화를 **요약·의도·감정·권장 조치**로 브리핑합니다(`assist` 기능).
- 브리핑 생성 실패 시 빈 값으로 처리되어 상담에는 지장이 없습니다.

### 6.4 답변 전송 (검열 필수)
- 상담원이 보내는 메시지도 **검열 게이트를 통과**해야 합니다.
- 검열에 차단되면 전송이 거부됩니다(오류 코드 `MODERATION_BLOCKED`). 표현을 수정해 다시 시도하세요.

### 6.5 대화 종료 (End)
- 종료하면 대화 상태가 `ended`가 되고 활성 배정이 해제됩니다.

### 6.6 상담원 프로필·통계
- 프로필: 사용 언어, 스킬, 동시 상담 최대 수(`max_concurrent`, 기본 3), 상태(online/offline 등).
- 일별 통계(`agent_daily_stats`)를 테넌트 스코프로 조회할 수 있습니다.

## 7. 고객정보 연동

**위치**: 고객(`/customers`)

### 7.1 상담원이 확인할 수 있는 정보
고객 목록·상세에서 다음 필드를 확인합니다.

| 필드 | 출처 | 설명 |
|---|---|---|
| `email`, `name` | Shopify / 세션 | 고객 식별 |
| `shopifyCustomerId` | Shopify | Shopify 고객 ID |
| `tier` | 내부 | guest / subscriber / regular |
| `shopifyTier` | Shopify | Shopify 기준 등급 |
| 주문 캐시 | Shopify/Odoo | 주문·배송 상태(내부 상태: paid/preparing/shipping/delivered) |

- 이메일로 검색할 수 있고, 목록은 테넌트 스코프로 필터링됩니다.
- 고객은 이메일 기준으로 조회-또는-생성되며, Shopify 고객 ID가 함께 캐시됩니다.

### 7.2 연동 상태 (정직한 표기)
| 연동 | 상태 | 설명 |
|---|---|---|
| **Shopify 고객·주문 캐시** | ✅ 구현됨 | 고객/주문 도메인에서 캐시·조회 |
| 배송 조회 · Fulfillment 웹훅 | ✅ 구현됨 | 주문 상태 매핑(POL-014) |
| **Odoo 고객/주문 연동** | 🟡 연동 상태 추적만 | `integration_status`(shopify/fulfillment/klaviyo/odoo/google_drive)와 테넌트별 자격증명(AES-256-GCM) 스키마는 준비됨. **실시간 동기화 로직은 로드맵**이며, 현재 주문 캐시 테이블은 Shopify/Odoo 양쪽 소스를 담도록 설계됨 |
| Klaviyo · Google Drive | 🟡 상태 추적 | 연동 상태 관리 대상 |

- 연동 상태는 관리자에서 `integrations/status`로 이름·상태·마지막 동기화 시각을 확인/갱신합니다.

> 요약: 현재 상담원이 콘솔에서 보는 고객정보는 **Shopify 소스가 실동작**하며, **Odoo는 스키마·자격증명·상태추적이 준비된 단계**입니다. Odoo 실데이터 동기화는 다음 단계 과제입니다.

## 8. 지식베이스(KB) 관리

**위치**: 지식(`/knowledge`)

- 지식 소스(`knowledge_sources`)와 문서(`kb_documents`), 게시글(`kb_board_posts`), 파일(`kb_files`)을 관리합니다.
- 문서는 `active` 플래그로 RAG 검색 포함 여부를 제어합니다. **비활성 문서는 답변 근거로 쓰이지 않습니다.**
- 소스 우선순위: **Knowledge Store > Google Drive**.
- 검색 적중률을 위해 제목·본문에 고객이 실제 사용하는 키워드를 포함하세요(현재 키워드 검색 기반).

**운영 팁**
- 정책 변경 시 관련 문서를 갱신하고, 구버전은 `active=0`으로 내려 혼선을 방지하세요.
- 카테고리를 활용해 배송/환불/상품 등으로 문서를 분류하면 답변 컨텍스트가 깔끔해집니다.

## 9. 검열(Moderation) 정책

- **모든 AI·상담원 발신 메시지**는 `ModerationService.moderate()`를 반드시 통과합니다(우회 불가, FR-069/POL-020).
- 오류 발생 시 기본값은 **차단(fail-safe = block)**입니다.
- 결정값: 허용 / 차단(BLOCKED). 차단 시 AI 답변은 노출되지 않고 상담원 인계로 전환되며, 상담원 메시지는 전송이 거부됩니다.
- 검열 규칙은 AI 설정 화면의 "검열 규칙" 섹션에서 관리합니다(콘텐츠 필터 규칙 추가/삭제).
- 모든 검열 결정은 `moderation_logs`에 기록됩니다.

## 10. 캠페인(발송) 관리

**위치**: 캠페인(`/campaigns`) · **권장 권한**: 마스터/디렉터 (operations)

대상 고객군에게 메시지를 발송하는 **아웃바운드 캠페인**을 만들고 발송합니다.

| 필드 | 설명 |
|---|---|
| `name` | 캠페인 이름 |
| `channel` | 발송 채널 (기본 `email`) |
| `message` | 발송 메시지 본문 |
| `status` | `draft`(초안) → `scheduled`(예약) → `sent`(발송됨) |
| `audienceSize` | 대상 고객 수 |
| `sentCount` | 발송 건수 |

- 새 캠페인은 `draft` 상태로 생성되며, 목록에서 **발송**을 실행하면 상태가 `sent`로 바뀝니다.
- 캠페인 목록·발송은 테넌트 스코프로 관리됩니다.

> 🟡 세분화된 오디언스 타게팅·예약(scheduled) 자동 발송 UI는 확장 로드맵입니다. 현재는 생성 → 즉시 발송 흐름이 기본입니다.

## 12. FAQ / 문제 해결

**Q. 봇이 계속 상담원에게 넘깁니다.**
KB에 관련 문서가 없거나 비활성이라 신뢰도가 낮은 경우가 많습니다. 해당 주제의 KB 문서를 등록·활성화하고, 고객이 쓰는 키워드를 본문에 포함하세요.

**Q. 위젯에 시나리오 버튼이 안 보입니다.**
저장된 버튼이 모두 `enabled=false`이거나 조회가 실패하면 기본 6버튼으로 폴백됩니다. AI 설정에서 최소 하나 이상 활성화됐는지 확인하세요.

**Q. 상담원 답변이 전송되지 않습니다.**
검열 게이트에 차단됐을 수 있습니다(`MODERATION_BLOCKED`). 표현을 완곡하게 바꿔 다시 시도하세요.

**Q. 인계 임계값(0.45)을 바꾸고 싶습니다.**
현재는 코드 상수입니다. 화면 설정은 로드맵이며, 변경은 개발팀에 요청하세요.

**Q. Odoo 고객정보가 안 보입니다.**
Odoo 실시간 동기화는 아직 로드맵 단계입니다. 현재 콘솔의 고객정보는 Shopify 소스 기준으로 표시됩니다.

**Q. AI 키가 없는데 동작합니다.**
스텁 어댑터가 키 없이 응답을 생성합니다(개발/폴백용). 실서비스 품질을 위해 Anthropic 등 실엔진 키를 설정하세요.

**Q. 고객이 알림을 안 받는다고 합니다.**
`in_app`은 항상 켜져 있지만 `email`/`sms`/`web_push`는 고객이 환경설정에서 카테고리별로 꺼둘 수 있습니다. CCPA 옵트아웃을 누르면 in_app 외 전 채널이 꺼집니다. 고객의 수신 환경설정을 확인하세요.

**Q. 재입고 알림은 로그인해야 하나요?**
아니요. 품절 상품 재입고 알림은 게스트도 구독할 수 있습니다(`POST /restock/subscribe`).

**Q. 캠페인을 예약 발송하고 싶습니다.**
현재는 생성 후 즉시 발송이 기본입니다. `scheduled` 상태·자동 예약 발송 UI는 로드맵입니다.
