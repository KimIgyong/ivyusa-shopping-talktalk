# IVY TalkTalk — User Guide (유저 그룹 설정·사용 매뉴얼)

대상: 테넌트 유저(Master / Director / Manager / Staff). 시스템 어드민/고객용은 별도. 본 매뉴얼은 **콘솔 접속 → 마스터 초기 설정 → 직급·직무별 일상 사용** 순서.

---

## 1. Getting Started (시작하기)

- **접속**: 관리자 콘솔(User Console)에 회사 이메일로 로그인. 최초 비밀번호 설정/2단계 인증 권장.
- **역할 확인**: 우상단 프로필에서 내 **직급(Master/Director/Manager/Staff)** 과 **직무 라벨(상담/회계/운영)** 확인. 메뉴는 보유 권한만 표시됩니다(없는 메뉴는 숨김).
- **권한 원칙**: 보이는 기능 = 직급 × 라벨. 필요한 메뉴가 안 보이면 Master/Director에게 라벨·등급 요청.

---

## 2. Master — Tenant Setup Guide (마스터 초기 설정 가이드)

Master는 테넌트(스토어) 운영을 위한 모든 설정을 담당합니다. **권장 순서:**

### 2.1 외부 연동 (설정 → 테넌트 설정·외부연동, SCR-207A) — Master 전용
1. **Shopify 연결**: 스토어 도메인 입력 → 인증/토큰 발급 → 주문·고객·웹훅 권한 확인. 연결 상태 "연결됨" 확인.
2. **Fulfillment 연결**: 배송 상태 웹훅 등록(배송 추적·알림 기반).
3. **Odoo 연결(선택)**: JSON-RPC 자격증명 입력 → 상품·재고 동기화 범위 선택.
4. **Klaviyo 연결(선택)**: API 키 입력 → 캠페인·세그먼트 사용(마케팅 알림).
5. **Google Drive 연결(선택)**: 보조 지식 폴더 연결 → KB 동기화.
- 자격증명은 저장 후 마스킹 표시. 연결 오류 시 상태 알림. 모든 변경은 감사 기록됩니다.

### 2.2 유저 초대·등급·라벨 (설정 → 유저·라벨 관리, SCR-207)
1. **유저 초대**: 이메일로 초대 → 직급 지정(Director/Manager/Staff).
2. **등급(직급) 조정**: 유저별 직급 변경(Master 전용).
3. **직무 라벨 부여**: 상담/회계/운영 다중 부여(Director도 부여 가능).
4. **라벨 편집(이름 수정/추가)**: Master 전용. 라벨 이름을 바꿔도 내부 권한 코드는 유지됩니다.

### 2.3 테넌트 정책값 (설정 → 테넌트 설정)
- 환불 기간, 반품 조건(예: 수령 후 7일·미개봉), 보증, 리뷰 요청 N일, 제휴 적립률 등 테넌트 정책값 입력. (미입력 시 글로벌 기본값 상속.)

### 2.4 AI 설정 (설정 → AI 설정, SCR-105) — Master/Director
- **Bot 페르소나**: 인사말·톤·언어 기본값(EN/ES/KO).
- **Rules**: 응답 규칙·금칙·에스컬레이션 임계치.
- **시나리오 버튼**: 위젯 메뉴(배송/취소환불/Product Help/Contact/Affiliate) 구성.

### 2.5 지식 소스 (설정 → AI 설정 → Knowledge Source, SCR-105K) — Master
RAG가 참조할 지식을 3가지 방식으로 등록·관리합니다. **AI는 "지정(designated)"된 활성 소스만 참조합니다.**
1. **게시판(Board)**: 글 작성 + **파일 첨부**. 공지/정책/FAQ 등을 글로 관리. 글/첨부가 지식화.
2. **자료실(File Repository)**: PDF/DOCX/TXT 등 **파일 직접 업로드**(드래그). 상태(임베딩 완료/대기) 확인.
3. **구글드라이브 연동**: 폴더 연결 → 동기화(주기/지금 동기화). 보조 지식원.
- 소스별 **활성/비활성**, **AI 참조 지정** 토글. 내용 수정/삭제 시 자동 **재임베딩**.
- 미지정/비활성 소스, 외부 임의 출처는 AI가 참조하지 않습니다(테넌트 격리).

### 2.6 유저 초대 흐름 (참고)
- 유저·라벨 관리에서 이메일 등록 → 시스템이 **임시 비밀번호 메일** 발송.
- 초대받은 유저는 최초 로그인 시 임시비번 인증 → **사용할 비밀번호로 변경(강제)** 후 이용.
- 임시비번 만료(기본 72시간) 시 재발송.

> 체크리스트: ☐ Shopify 연결 ☐ 배송 연동 ☐ 유저 초대(임시비번) ☐ 라벨 부여 ☐ 정책값 ☐ 지식 소스(게시판/자료실/GDrive) 등록·지정 ☐ 시나리오 ☐ 위젯 Theme App Embed 활성(테마 편집기).

> 최초 시드 계정: 시스템 어드민 `admin@amoeba.group`, 테넌트 Master `dev@amoeba.group` (테넌트 `ivyusa`). 최초 로그인 시 비밀번호 변경 필수.

## 3. Daily Use by Role (직급·직무별 일상 사용)

### 3.1 상담 (Consult 라벨 · 모든 직급)
- **실시간 채팅(SCR-102)**: 좌측 세션 목록(상담사 연결/AI 응답 중/대기/종료) → 대화 선택 → **개입 중**으로 인계받기 → 응답 → **종료**. 우측 **AI 상황 브리핑**(요약·고객정보·의도·감정·추천 액션) 참고.
- **이관**: AI↔상담사 전환 시 맥락 유지. 다른 담당자에게 재배정 가능(Manager+).
- **상담 이력**: 과거 대화 검색·열람(직무 연관 범위).

### 3.2 운영 (Operations 라벨)
- **주문/배송**: 주문 상태 조회, 배송 추적, 취소/환불 가이드 처리.
- **상품/재고**: 카탈로그·재고 확인(Odoo/Shopify), 재입고 알림 관리.
- **알림/캠페인**: 이벤트·쿠폰 캠페인 작성 → 세그먼트(Klaviyo) 선택 → **발송(Manager 이상)**. 옵트아웃 자동 제외.
- **리뷰**: 리뷰 요청(N일) 모니터링, 리뷰 응대.

### 3.3 회계 (Accounting 라벨)
- **결제/환불**: 결제 내역·환불 처리 뷰.
- **재무 리포트**: 기간별 매출/환불 리포트 열람·내보내기.

### 3.4 관리 (Manager / Director)
- **대화 배정·이관 관리**, **팀 유저 관리(Manager: 팀 한정)**, **분석/리포트**(AI 해결률·미해결 패턴·인기 질문), **캠페인 발송**.
- Director: 하위 유저 초대·라벨 부여, AI 설정 운영(자격증명·등급조정·라벨수정은 Master 전용).

## 5. FAQ (자주 묻는 질문)

| 질문 | 답변 |
|------|------|
| 메뉴가 안 보여요 | 직급 또는 직무 라벨 권한이 없을 수 있습니다. Master/Director에게 요청하세요. |
| 외부 연동은 누가 하나요 | **Master만** 외부연동 자격증명을 입력/변경할 수 있습니다(SCR-207A). |
| 직급을 바꾸려면 | **Master**가 유저·라벨 관리에서 등급을 조정합니다. |
| 라벨 이름을 바꾸고 싶어요 | **Master**가 라벨 편집에서 이름을 변경합니다(권한은 그대로 유지). |
| 캠페인을 못 보내요 | 발송은 **Manager 이상** 권한입니다(Staff 불가). |
| 고객 개인정보가 안 보여요 | 고객 인증(Auth Gate) 전이거나 직무 범위 밖일 수 있습니다(CCPA·최소권한). |

## 6. Troubleshooting (문제 해결)

| 증상 | 원인 | 해결 |
|------|------|------|
| 연동 "오류" 표시 | 자격증명 만료/권한 부족 | Master가 재연결(SCR-207A), 권한 스코프 확인 |
| AI 답변 품질 저하 | KB 미갱신 | AI 설정 → Knowledge 갱신(재임베딩) |
| 알림 미수신(고객) | 채널/옵트아웃 | 고객 알림 설정·CCPA opt-out 확인 |
| 배송 상태 안 맞음 | Fulfillment 웹훅 지연 | 연동 상태/last-sync 확인, 재시도 대기 |
| 권한 변경 미반영 | 캐시/세션 | 재로그인 후 확인(권한은 deny-by-default) |
