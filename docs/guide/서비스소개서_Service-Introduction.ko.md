# IVY USA 채팅·고객지원 위젯 — 서비스 소개서

> 버전 1.0.0 · 작성일 2026-07-01 · 대상: 경영진 · 도입 검토자 · 파트너
> 문서 성격: 서비스 개요와 설계 개념 소개 (실제 구현 상태 기준)

## 1. 한눈에 보는 서비스

IVY USA 채팅·고객지원 위젯은 **Shopify 스토어프론트에 탑재되는 멀티테넌트 AI 상담 위젯**입니다. 네이버 톡톡과 유사한 사용자 경험을 제공하며, 고객 문의를 3단계로 자동·수동 응대합니다.

1. **시나리오 챗봇** — 배송·취소/환불·상품·제휴 등 정형 문의를 버튼 메뉴 기반으로 즉시 안내
2. **RAG 지식기반 AI 챗봇** — 시나리오로 해결되지 않는 문의를 테넌트 지식베이스(KB)에 근거해 답변하고 출처를 인용
3. **상담원 인계(에스컬레이션)** — AI 신뢰도가 낮거나 지식 범위를 벗어나면 자동으로 사람 상담원에게 연결

여기에 **테넌트 운영자 콘솔**(라이브챗·설정·지식·고객 관리)과 **플랫폼 관리자 콘솔**(테넌트·AI엔진·감사)이 함께 제공됩니다. 최초 테넌트는 IVY USA입니다.

## 2. 핵심 가치 (Core Values)

| 가치 | 설명 |
|---|---|
| **근거 기반 AI** | RAG는 지정·활성화된 지식 소스에서만 답변하고 출처를 인용합니다. 근거가 부족하면 임의로 답하지 않고 상담원에게 넘깁니다. |
| **안전 우선** | 모든 AI·상담원 발신 메시지는 우회 불가능한 검열(Moderation) 게이트를 통과합니다. 오류 시 기본값은 "차단"(fail-safe). |
| **테넌트 격리** | 데이터·설정·자격증명·AI 라우팅이 테넌트별로 분리됩니다. 교차 테넌트 데이터 노출을 원천 차단합니다. |
| **컴플라이언스** | CCPA/CPRA + GDPR 대응 자세 — 동의·옵트아웃·암호화·감사 로그를 내장합니다. |
| **교체형 AI** | 기능(chat/rag/summary/assist/moderation)별로 AI 엔진(Anthropic 등)을 선택할 수 있고, 장애 시 스텁으로 자동 강등됩니다. |

## 3. 대응 흐름 개념도

```
스토어프론트 고객 (위젯)
      │
      ▼
[비로그인 진입] ── 시나리오 챗봇 (버튼 메뉴)
      │              배송 · 취소/환불 · 상품안내 · 제휴 · 내 주문 · 상담원 연결
      │
      ├─(주문 등 개인정보 필요)──▶ 본인확인 게이트 (로그인 / 비회원 주문조회)
      │
      ▼
[정형 답변으로 불충분] ── RAG 지식기반 AI 챗봇
      │              KB 검색 → 근거 기반 생성 → 출처 인용 → 검열 게이트
      │
      ├─(신뢰도 < 0.45 또는 검열 차단 또는 고객 요청)
      ▼
[상담원 인계] ── 상담원 콘솔 (대기열 → 인수 → AI 브리핑 → 검열된 답변)
```

## 4. 시스템 아키텍처 (설계 개념)

**모노레포(Turborepo)** 구조로 3개 앱과 2개 공유 패키지로 구성됩니다.

- `apps/widget` — Shopify 임베드형 고객 위젯 (React 18)
- `apps/web` — 운영자·관리자 SPA 콘솔 (React 18 + react-router)
- `apps/api` — 백엔드 API (NestJS 10, `/api/v1`)
- `packages/types`, `packages/common` — 공유 enum·응답 규격·RBAC 매트릭스·유틸

**기술 스택**

| 구분 | 기술 |
|---|---|
| 프론트엔드 | React 18 · TypeScript · Vite · TailwindCSS · Zustand · TanStack React Query · react-i18next |
| 백엔드 | NestJS 10 · TypeORM · MySQL 8 · Redis 7 · RabbitMQ · JWT · AES-256-GCM |
| AI | 교체형 AI 게이트웨이 (스텁 / Anthropic 어댑터) |
| 다국어 | 영어(en, 기본) · 스페인어(es) · 한국어(ko) |

**설계 원칙**: Clean Architecture + DDD, 도메인 모듈형, API 우선, 4계층(Controller → Service → Entity → Repository), `tenant_id` 기반 멀티테넌시. 컨트롤러는 비즈니스 로직을 갖지 않고, 서비스가 로직과 테넌트 스코프를 담당합니다.

```
Shopify 스토어프론트 ─▶ apps/widget ─┐
운영자/관리자        ─▶ apps/web    ─┼─▶ apps/api (NestJS)
                                      │     ├─ MySQL 8 (TypeORM)
                                      │     ├─ Redis 7 (캐시/세션/미확인)
                                      │     ├─ RabbitMQ (알림/CJM/로그 이벤트)
                                      │     ├─ AI 게이트웨이 → 스텁 | Anthropic
                                      │     └─ 검열 게이트 (fail-safe 차단)
외부 연동: Shopify · Fulfillment 웹훅 · Klaviyo · Odoo · Google Drive
          (테넌트별 자격증명, AES-256-GCM 암호화)
```

## 5. 주요 기능 개요

**고객 위젯**
채팅 위젯, 알림 센터(미확인 배지), 시나리오 버튼 메뉴, 세션·다국어, RAG 상품 Q&A, 주문·배송 조회, 리뷰·제휴·재입고/구독 신청, 상담원 인계.

**운영자 콘솔 (테넌트)**
- 대시보드 — 상담 지표 개요
- 라이브챗 — 대기열, 대화 인수, AI 브리핑, 검열된 답변, 종료
- 상담 이력 — 상태·에스컬레이션 필터 조회
- AI 설정 — 봇 페르소나, 응답 규칙, 시나리오 버튼, 기능별 AI 엔진, 검열 규칙
- 지식(KB) — 지식 소스·문서 관리 (Knowledge Store 우선, Google Drive 보완)
- 고객 — Shopify 고객 캐시·등급 조회
- 캠페인 · 사용자/권한 · 환경설정

**플랫폼 관리자 콘솔**
전체 개요, 테넌트 관리, AI 엔진 관리, 감사 로그.

**공통 기반**
RBAC(직급 master/director/manager/staff × 라벨 consult/accounting/operations), 표준 응답 래핑, 오류 코드 체계(Exxxx), 감사 로그, PII 마스킹.

## 6. AI 통합 개념

`AiGatewayService`가 `(테넌트, 기능)` 조합에 대해 사용할 엔진을 `tenant_ai_settings → ai_engines`로 해석하고, 프로바이더 어댑터(스텁 | Anthropic)로 위임한 뒤 결과를 정규화합니다. 장애 시 스텁으로 자동 강등되어 서비스가 중단되지 않습니다.

적용 영역: RAG 답변(출처 인용), 의도 분류, 상담원 AI 브리핑, 응답 검열(LLM 컨텍스트 분류), 재작성(rephrase). **모든 AI 발신은 검열 게이트를 반드시 통과합니다.**

## 7. 보안·컴플라이언스

- **인증**: 관리자·운영자는 JWT(액세스 15분/리프레시 7일). 고객 위젯은 불투명 세션 토큰 + Shopify 고객 인증/비회원 주문조회(15분당 최대 5회, Redis 레이트리밋).
- **암호화**: 비밀번호 bcrypt(cost 12), 외부 자격증명 AES-256-GCM.
- **개인정보**: 로그 PII 마스킹, 감사 로그, 보존/파기 정책(기본 365일), Shopify GDPR 웹훅(`customers/data_request`, `customers/redact`, `shop/redact`, HMAC 검증), DSAR 내보내기/삭제·CCPA "판매·공유 거부" 옵트아웃.
- **멀티테넌시**: 전 테넌트 스코프 테이블에 `tenant_id` 적용 + AsyncLocalStorage 기반 테넌트 컨텍스트 + TypeORM 구독자 자동 스탬핑.

## 8. 현재 구현 상태 (정직한 스냅샷)

> 도입 검토에 필요한 실제 상태입니다. 표기는 **구현됨 / 설계·연동 준비됨 / 로드맵**으로 구분합니다.

| 영역 | 상태 | 비고 |
|---|---|---|
| 시나리오 버튼 챗봇 + 관리자 설정 | ✅ 구현됨 | 6개 기본 액션, 테넌트별 커스터마이즈 |
| RAG 답변 + 출처 인용 + 신뢰도 산정 | ✅ 구현됨 | 검색기는 **키워드 기반**(벡터 스토어 대체) |
| 검열 게이트 (AI·상담원 발신) | ✅ 구현됨 | fail-safe 차단, 단위테스트 포함 |
| 상담원 인계·콘솔·AI 브리핑 | ✅ 구현됨 | 대기열/인수/종료/통계 |
| 봇 페르소나·응답 규칙·기능별 엔진 설정 | ✅ 구현됨 | AI 설정 화면 |
| Shopify 고객 캐시·주문 캐시·배송 조회 | ✅ 구현됨 | 고객/주문 도메인 |
| 멀티테넌시·RBAC·감사·개인정보(GDPR/DSAR) | ✅ 구현됨 | 표준 준수 감사 완료 |
| AI 엔진 어댑터 | ✅ 스텁 / ✅ Anthropic | 키 없이 스텁 구동 |
| **RAG 벡터 검색(임베딩)** | 🟡 로드맵 | 현재 키워드 검색이 대체 |
| **Odoo 커넥터 실연동** | 🟡 연동 상태 추적만 | `integration_status`·자격증명 스키마는 준비, 실 동기화 로직 로드맵 |
| **AI 토큰 쿼터(E4010/E4011)** | 🟡 예약됨 | 오류코드만 예약, 테넌트별 집계 로드맵 |
| **스테이징/운영 배포** | ✅ Docker 산출물 | 도메인·운영 파이프라인 TBD |
| 테스트 커버리지 | 🟡 부분 | 단위테스트 46개, e2e는 로드맵 |

**요약**: 3단계 응대 흐름(시나리오 → RAG → 상담원), 관리자 설정, 검열, 멀티테넌시·컴플라이언스는 동작합니다. RAG의 벡터 검색 고도화, Odoo 실연동, 토큰 쿼터 집계, e2e 테스트가 다음 단계 과제입니다.

## 9. 도입 시 이점 요약

- 정형 문의는 봇이 즉시 처리해 **상담원 업무량 감소**
- RAG가 **근거 기반**으로 답해 오답·환각 위험을 낮추고 출처로 신뢰 확보
- 저신뢰·위험 답변은 자동으로 사람에게 넘겨 **품질·안전 유지**
- 멀티테넌트 구조로 **여러 Shopify 스토어 확장** 가능
- 개인정보·검열 정책 내장으로 **컴플라이언스 부담 완화**
